# Contributing to Read the Room

Thanks for your interest in Read the Room! This is a solo-maintained project, so
contributions look a little different than larger open-source efforts. Here's how
you can get involved.

## Bug Reports

Bug reports are welcome and encouraged. Please
[open an issue](https://github.com/soud/readtheroom-app/issues/new?template=bug_report.md)
with:

- Steps to reproduce
- Expected vs. actual behavior
- Device, OS version, and app version
- Screenshots or screen recordings if applicable

## Feature Requests and Discussion

Have an idea? Open a
[feature request issue](https://github.com/soud/readtheroom-app/issues/new?template=feature_request.md)
or start a discussion. Even if a feature doesn't get built right away, the
conversation helps shape the project's direction.

There's also a Feedback page within the app to brainstorm ideas and upvote feature requests that is actively monitored.

## Security Vulnerabilities

If you discover a security vulnerability, please report it privately. **Do not
open a public issue.** See [SECURITY.md](SECURITY.md) for instructions.

## Code Contributions

Read the Room is **not currently accepting pull requests**. This is a developer bandwidth
decision, not a quality judgment — reviewing and merging external code takes time
that's better spent on the app right now.

This may change in the future. Star or watch the repo to stay updated please!

If you've found a bug and want to point out the fix, you're welcome to reference
specific lines or share a diff in an issue — that's genuinely helpful even
without a PR.

## Code of Conduct

Be respectful and constructive. Harassment, spam, and bad-faith engagement will
result in being blocked.
